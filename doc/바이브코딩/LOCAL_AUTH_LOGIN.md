# 로컬 인증(아이디/비밀번호) 로그인 문서

이 문서는 현재 프로젝트에 추가된 **로컬 인증 로그인** 기능의 동작 메커니즘과 사용 방법을 설명합니다.

## 목적

브라우저 기반으로 동작하는 개인용 프로젝트에서, **나 혼자라도 비밀번호를 입력해야만** 전체 기능(지도/할 일/식물/프로젝트 등)에 접근할 수 있도록 합니다.

## 동작 메커니즘(핵심)

### 1) 자격증명(아이디/비밀번호) 저장 방식

- 최초 1회: `아이디 + 비밀번호`를 입력하면, 비밀번호는 **해시로 변환(PBKDF2)** 된 뒤 `localStorage`에 저장됩니다.
- 이후 로그인: 입력한 비밀번호를 같은 방식으로 해시화하여 저장된 해시와 비교합니다.
- 세션: 로그인 성공 시 `localStorage`에 “로그인 상태(사용자명)”를 저장합니다.

구현 위치:
- 해시/저장 로직: `src/shared/lib/localAuth.js`
- 인증 상태 관리: `src/app/providers/AuthContext.jsx`
- 라우트 가드: `src/app/providers/RequireAuth.jsx`

### 2) 비밀번호 해시 알고리즘(PBKDF2)

- 해시: `PBKDF2` + `SHA-256`
- salt: 랜덤 생성(16바이트)
- iteration(반복 횟수): `120000`
- 저장값에는 `salt`와 `passwordHash(해시)`가 포함됩니다.

즉, 비밀번호 원문은 저장되지 않습니다.

## 사용 방법

### 1) 최초 계정 설정(처음 접속 시)

1. 앱 접속
2. 로그인 화면에서 **“계정 설정”** 모드로 표시되며,
   - `아이디`
   - `비밀번호`
   - `비밀번호 확인`
3. 입력 후 버튼을 누르면 계정이 저장되고 자동으로 로그인됩니다.

### 2) 로그인(이후 접속 시)

1. 앱 접속
2. 로그인 화면에서
   - `아이디`
   - `비밀번호`
3. 제출하면 저장된 해시와 비교 후 로그인됩니다.

### 3) 로그아웃

- 상단 바(`AppBar`)의 로그아웃 버튼 또는 좌측 네비 레일/하단 메뉴의 로그아웃 버튼을 누르면 세션이 해제되고 `#/login`으로 이동합니다.

### 4) 계정 재설정(비밀번호 변경/초기화)

- 로그인 화면 하단에 **“계정 재설정”** 버튼이 있습니다.
- 누르면 다시 “계정 설정” 모드로 전환되어 새 비밀번호로 자격증명을 덮어씁니다.

## 저장되는 데이터(브라우저 관점)

저장 키(예시):
- `digital-garden.auth.credentials.v1` : 아이디/솔트/비밀번호 해시
- `digital-garden.auth.session.v1` : 로그인 사용자명(세션)

## 보안 주의사항(중요)

이 방식은 “서버 인증”이 아니라 **브라우저 로컬스토리지 기반 개인용 보호**입니다.

- 같은 브라우저/같은 프로필을 사용하는 사용자는 `localStorage`를 조작할 수 있습니다.
- 따라서 이 기능은 “완전한 보안”이 아니라, **우연한 접근/노출을 줄이기 위한 최소 보호**로 이해하는 것이 좋습니다.

## 문제 해결

- 로그인이 계속 실패하면:
  - 로그인 화면에서 **계정 재설정** 후 다시 입력해 보세요.
  - 그래도 안 되면 브라우저의 개발자 도구에서 `localStorage`를 확인하고 관련 키를 삭제하세요.
